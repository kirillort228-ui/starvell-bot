# Starvell Notify Bot

Telegram-бот для уведомлений о новых сообщениях в Starvell.

Бот работает через Next.js JSON-запрос Starvell:

```text
GET https://starvell.com/_next/data/{build_id}/chat.json
```

Он получает список чатов, смотрит `unreadMessageCount`, `user.unreadChatIds` и `lastMessage.id`, а потом присылает уведомление в Telegram с кнопкой «Открыть чат».

## Возможности

- Подключение Starvell аккаунта через Cookie без обязательного прокси.
- Прокси можно добавить позже отдельно для конкретного аккаунта.
- Поддержка прокси:
  - `http://ip:port`
  - `https://ip:port`
  - `socks5://ip:port`
  - `socks5://login:password@ip:port`
- Проверка новых сообщений каждые 60 секунд.
- Ссылка на конкретный чат: `https://starvell.com/chat/{chat_id}`.
- SQLite база данных.
- Включение/выключение уведомлений для каждого аккаунта.

## Как добавить аккаунт

1. Открой бота в Telegram.
2. Нажми `/start`.
3. Нажми `🔐 Мои аккаунты`.
4. Нажми `➕ Добавить аккаунт`.
5. Отправь Cookie от Starvell.

Готово: аккаунт добавится сразу без прокси. Старые сообщения не будут отправлены повторно.

## Как добавить прокси к аккаунту

1. Нажми `🔐 Мои аккаунты`.
2. Под нужным аккаунтом нажми `🌐 Прокси аккаунта`.
3. Отправь прокси в формате `http://ip:port`, `https://ip:port`, `socks5://ip:port` или `socks5://login:password@ip:port`.
4. Чтобы убрать прокси, отправь `-`.

## Где взять Cookie

В Chrome:

1. Открой `starvell.com` и войди в аккаунт.
2. Нажми `F12`.
3. Открой `Network / Сеть`.
4. Нажми `Fetch/XHR`.
5. Открой запрос `chat.json`.
6. В `Headers / Заголовки` найди `Cookie`.
7. Скопируй значение Cookie.

Cookie — это почти как пароль. Не публикуй его, не отправляй в чужие чаты и не выкладывай на GitHub.

## Важные замечания

- При добавлении аккаунта бот сохраняет текущие `lastMessage.id`, чтобы не отправлять старые сообщения.
- Уведомление приходит только при непрочитанном чате.
- Если Starvell изменит структуру сайта или endpoint, клиент может потребовать обновления.
- Cookie может устаревать. Тогда нужно удалить аккаунт в боте и добавить свежий Cookie.

## Пополнение баланса бота

В разделе **👤 Профиль** появилась кнопка пополнения. Сейчас реализовано ручное пополнение:

1. Пользователь нажимает **Пополнить баланс бота**.
2. Пользователь вводит любую сумму в рублях, например `150` или `349.50`. Также доступны быстрые кнопки 100/300/500/1000 ₽.
3. Бот создаёт заявку на пополнение.
4. Пользователь оплачивает по реквизитам из `.env`.
5. Администратор подтверждает заявку кнопкой в Telegram.
6. Баланс пользователя в боте увеличивается.

В `.env` можно указать:

```env
ADMIN_ID=твой_telegram_id
TOP_UP_PAYMENT_DETAILS=перевод на карту 0000 0000 0000 0000, комментарий — ID заявки
```

Если `ADMIN_ID` не указан, бот всё равно создаст заявку, но администратору не придёт кнопка подтверждения.

## Проверка аккаунта не зависает

При добавлении аккаунта проверка Cookie ограничена 30 секундами. Если Starvell не отвечает, Cookie устарел или IP хостинга заблокирован, бот покажет ошибку вместо бесконечного сообщения `Проверяю аккаунт`.


## Топ продавцов без отдельной страницы

В STARVELL может не быть отдельной публичной страницы топа продавцов. Поэтому бот теперь собирает топ самостоятельно:

1. открывает публичные категории Starvell, например `/roblox/packages`;
2. находит продавцов, рейтинг, отзывы и процент выполнения;
3. удаляет дубли продавцов;
4. сортирует по количеству отзывов и рейтингу;
5. показывает топ в Telegram.

Если Railway долго собирает топ или Starvell блокирует IP хостинга, добавьте прокси к аккаунту.

## Обновление под offerPublicId

Бот подготовлен под изменение API Starvell: во всех будущих операциях с предложениями используется `offerPublicId` / UUID вместо старого числового `offerId`.

Что изменено:
- добавлен `extract_offer_public_id()`;
- добавлен `ensure_offer_public_id()`;
- добавлены методы:
  - `get_offer_by_public_id()`;
  - `update_offer_by_public_id()`;
  - `partial_update_offer_by_public_id()`;
  - `raise_offer_by_public_id()`;
- числовой `offerId` больше не принимается для offer routes;
- добавлена команда `/api_check`.

Текущие уведомления, продажи, статистика, профиль и топ продавцов не используют маршруты `offers/{id}`, поэтому они не должны сломаться из-за перехода Starvell на UUID.


## Исправление общей статистики v3

Вкладка **📊 Статистика** теперь не показывает нули, если Starvell не отдал список заказов в `orders.json`, но в чатах есть системные уведомления `ORDER_COMPLETED`, `ORDER_REFUND`, `REVIEW_CREATED`.

Добавлено:
- строка версии `stats-notifications-orders-v3`;
- источник заказов и количество заказов в ответе Starvell;

- дополнительные варианты Next.js endpoint для продаж.

Если сумма всё равно 0, значит Starvell не отдал цены заказов в доступном endpoint; количество будет взято из уведомлений, а сумма появится только когда бот получит `orders` с `basePrice`.

## Реальная отправка сообщений Starvell

Подключён реальный endpoint из Network:

```text
POST https://starvell.com/api/messages/send
```

Payload:

```json
{
  "chatId": "...",
  "clientSocketId": "...",
  "content": "..."
}
```

Теперь через этот endpoint могут работать:
- `👋 Приветствие`;
- `🤖 Автоответчик`;
- `⏰ Напоминание о подтверждении` (если найден чат клиента).

Важно:
- `Приветствие` отправляется один раз в каждый чат.
- `Автоответчик` отправляется на новое непрочитанное обычное сообщение клиента, если задан текст автоответчика.
- Для остальных функций вроде автоподнятия/автовыставления/автоподтверждения ещё нужны их отдельные Network-запросы.

## Чат-автофункции v1

Через реальный endpoint `POST /api/messages/send` теперь работают чат-сценарии:

- `👋 Приветствие` — отправляется один раз в каждый новый чат.
- `🤖 Автоответчик` — отвечает на новое непрочитанное обычное сообщение клиента.
- `💬 Текст при игноре` — отправляется, если последнее непрочитанное сообщение клиента остаётся без ответа дольше задержки. По умолчанию задержка 60 минут. Изменить можно через БД/настройку `ignore_delay_minutes`.
- `💬 Текст после 5 звёзд` — отправляется при уведомлении `REVIEW_CREATED`, если рейтинг 5 или Starvell не отдал рейтинг в metadata.
- `💬 Текст при проблеме` — отправляется при уведомлениях с `REFUND`, `DISPUTE`, `PROBLEM`, `ISSUE`, `CANCEL`.
- `💬 Текст после вашего подтверждения` — отправляется при событиях доставки/подтверждения со стороны продавца, если Starvell отдаёт такой notificationType.
- `💬 Текст после подтверждения клиентом` — отправляется при `ORDER_COMPLETED` / подтверждении клиентом.

Добавлена команда `/chat_auto_debug`.
Версия: `chat-auto-events-v1`.

## Резервный buildId v7

При поиске товаров бот сначала пытается получить актуальный Next.js `buildId` с главной страницы Starvell.

Если главная страница недоступна из Railway, сработала защита Cloudflare или прокси не отвечает, бот использует резервный `buildId`:

`NzQuXbo4EDoGpiOH6ZurF`

Это позволяет продолжить перебор сохранённых категорий вместо остановки с ошибкой.
